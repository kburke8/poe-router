    **Step 3: Create `src/lib/gem-migration.ts`** with these exports:

    1. `resolveGemName(name: string): string` — Looks up name in gem-rename-map.json, returns mapped name or original.

    2. `migrateBuildGemNames(build: BuildPlan): BuildPlan` — Deep clones via JSON.parse(JSON.stringify(build)), then walks all 5 gem name sites calling resolveGemName on each. The 5 sites are:
       - `build.stops[].gemPickups[].gemName` (Site 1)
       - `build.stops[].droppedGems[]` (Site 2)
       - `build.linkGroups[].phases[].gems[].gemName` (Site 3)
       - `build.linkGroups[].phases[].gems[].alternatives[].gemName` (Site 5, recursive via helper)
       - `build.mulePickups[].gemName` (Site 4)
       Sets `migrated.updatedAt = new Date().toISOString()`.
       Uses a helper `migrateGemSlot(slot: GemSlot): void` that renames slot.gemName and recurses into slot.alternatives.

    3. `migratePresetGemNames(preset: RegexPreset): RegexPreset` — Deep clones, walks `preset.categories[].entries[]`, renames `entry.sourceName` if present. Does NOT touch `entry.pattern`. Sets `migrated.updatedAt`.

    4. `checkIfMigrationNeeded(builds: BuildPlan[], presets: RegexPreset[]): boolean` — Scans all gem name sites in builds and all sourceName fields in presets. Returns true if ANY value exists as a key in the rename map.

    5. `runGemMigrationIfNeeded(): Promise<void>` — The orchestrator:
       - Check `localStorage.getItem('gem-migration-v328')` — if `'done'`, return immediately
       - Load all data from Dexie: `db.builds.toArray()`, `db.regexPresets.toArray()`, `db.runs.toArray()`, `db.customGems.toArray()`, `db.customItems.toArray()`
       - Call `checkIfMigrationNeeded(builds, presets)` — if false, set localStorage flag and return
       - Create backup ExportData with all tables (using extended ExportData type with customGems/customItems)
       - Show toast via `toast('Pre-migration backup created for patch 3.28', { action: { label: 'Download Backup', onClick: () => downloadJson(backupData, 'poe-planner-pre-migration-backup.json') }, duration: 10000 })`
       - Migrate each build with `migrateBuildGemNames()` and `db.builds.put(migrated)` — wrap in try/catch per build. If ANY build fails, do NOT set the localStorage flag (retry on next load per user decision).
       - Migrate each preset with `migratePresetGemNames()` and `db.regexPresets.put(migrated)` — same error handling.
       - Only after ALL migrations succeed: `localStorage.setItem('gem-migration-v328', 'done')`

    Note: `runGemMigrationIfNeeded` uses browser APIs (localStorage, toast/DOM). The unit tests should only test the pure functions (resolveGemName, migrateBuildGemNames, migratePresetGemNames, checkIfMigrationNeeded). The orchestrator function is tested via the manual checkpoint in the verification step.

    **Important anti-patterns to avoid (from research):**
    - Do NOT migrate `RegexEntry.pattern` fields — only `sourceName`
    - Do NOT use Dexie version upgrade — use app-level localStorage flag
    - DO recurse into `GemSlot.alternatives` (the 5th site missed in the original 4-site spec)
    - Do NOT mark migration complete on partial failure
